Edit the Product model in `prisma/schema.prisma`. Add two new fields immediately after `processedData` (currently line 76) and before `detailPageUrl` (currently line 77):

```prisma
  draftContent   Json?    @map("draft_content")
  pipelineStep   String?  @map("pipeline_step")
```

Then add a new index in the index block (after `@@index([abcGrade])`, currently line 105):

```prisma
  @@index([pipelineStep])
```

Per D-01/D-02: `pipelineStep` is a separate nullable String column independent from `status`. Valid values are `null`, `content_ready`, `images_generating` — enforced at app level, NOT via DB enum (native PG enum forbidden per CLAUDE.md).

Per D-05 (Claude's discretion): `draftContent` is `Json?` with `@map("draft_content")`, matching the exact pattern of `rawData`/`processedData`. No `@default()` — null means "not yet generated."

Critical rules:
- `@map("draft_content")` is REQUIRED — without it, Python agents using asyncpg raw SQL with snake_case column names will fail
- `@map("pipeline_step")` is REQUIRED — same reason
- Do NOT add `@default(null)` — Prisma v7 does not accept this syntax; `?` suffix is sufficient
- Do NOT add `@default("")` to pipelineStep — empty string is not null; null means "no pipeline step in progress"
- Do NOT add `@default("{}")` to draftContent — would break frontend checks like `if (product.draftContent)`
- Do NOT modify `processedData` or `status` columns — out of scope
