**Method 1: `getReturnSummary(companyId: string, from: Date, to: Date)`**

Computes return rate by counting returns and orders in the same date range. Uses `Promise.all` to fan-out both counts concurrently.

```typescript
async getReturnSummary(companyId: string, from: Date, to: Date) {
  const [returnRows, orderRows] = await Promise.all([
    this.prisma.$queryRaw<{ count: number }[]>`
      SELECT COUNT(*)::int AS count
      FROM coupang_returns
      WHERE company_id = ${companyId}::uuid
        AND requested_at >= ${from}
        AND requested_at < ${to}
    `,
    this.prisma.$queryRaw<{ count: number }[]>`
      SELECT COUNT(*)::int AS count
      FROM coupang_orders
      WHERE company_id = ${companyId}::uuid
        AND ordered_at >= ${from}
        AND ordered_at < ${to}
    `,
  ]);

  const returnCount = Number(returnRows[0]?.count ?? 0);
  const orderCount = Number(orderRows[0]?.count ?? 0);
  const returnRate = orderCount > 0
    ? Math.round((returnCount / orderCount) * 10000) / 100
    : 0;

  return { returnCount, orderCount, returnRate };
}
```

CRITICAL: Return rate is `(returnCount / orderCount * 100)` rounded to 2 decimal places. Use `Math.round(x * 10000) / 100` to avoid floating point drift. Guard against division by zero.
CRITICAL: Use `requested_at` for returns (not `created_at`) — this is when the return was requested by the customer.
CRITICAL: Use `ordered_at` for orders — consistent with the trend endpoint.

**Method 2: `getReturnReasonBreakdown(companyId: string, from: Date, to: Date)`**

Groups returns by `cancelReasonCategory1`, counts each, returns sorted descending by count. Uses `$queryRaw`.

```typescript
async getReturnReasonBreakdown(companyId: string, from: Date, to: Date) {
  const rows = await this.prisma.$queryRaw<
    { reason: string | null; count: number }[]
  >`
    SELECT
      cancel_reason_category1 AS reason,
      COUNT(*)::int AS count
    FROM coupang_returns
    WHERE company_id = ${companyId}::uuid
      AND requested_at >= ${from}
      AND requested_at < ${to}
    GROUP BY cancel_reason_category1
    ORDER BY count DESC
  `;

  return rows.map((r) => ({
    reason: r.reason ?? '미분류',
    count: Number(r.count),
  }));
}
```

CRITICAL: NULL `cancelReasonCategory1` values are mapped to '미분류' (uncategorized) in the TypeScript map.
CRITICAL: `::int` cast on COUNT to avoid BigInt serialization.

**Method 3: `getReturnFaultSplit(companyId: string, from: Date, to: Date)`**

Groups returns by `faultByType` (CUSTOMER vs VENDOR), returns both counts.

```typescript
async getReturnFaultSplit(companyId: string, from: Date, to: Date) {
  const rows = await this.prisma.$queryRaw<
    { fault_type: string; count: number }[]
  >`
    SELECT
      fault_by_type AS fault_type,
      COUNT(*)::int AS count
    FROM coupang_returns
    WHERE company_id = ${companyId}::uuid
      AND requested_at >= ${from}
      AND requested_at < ${to}
    GROUP BY fault_by_type
  `;

  const result: Record<string, number> = {};
  for (const r of rows) {
    result[r.fault_type] = Number(r.count);
  }

  return {
    customer: result['CUSTOMER'] ?? 0,
    vendor: result['VENDOR'] ?? 0,
  };
}
```

CRITICAL: Returns `{ customer: number, vendor: number }` — not an array. Frontend can directly use these values.
CRITICAL: `::int` cast and `Number()` conversion for BigInt safety.
CRITICAL: Default to 0 when a fault type has no rows (e.g., if all returns are CUSTOMER, vendor should be 0, not undefined).
