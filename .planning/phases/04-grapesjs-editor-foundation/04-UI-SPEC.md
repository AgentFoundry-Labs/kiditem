## Interaction Contract

### Draft Entry Flow (EDIT-01, EDIT-02)

Trigger: User navigates to `/sourcing/[id]/editor` for a product with `status = 'draft'` (preview.template === null).

Expected sequence:
1. Loading screen appears immediately: `Loader2` spinner (32px) + "에디터를 준비하고 있습니다..." at 14px gray-400
2. Three parallel fetches complete: product detail, preview, templates-styles.css
3. Branch condition: `preview.template === null` → set `mode = 'grapes'`, `previewData = placeholderDetailPageData`, `templateConfig = getTemplate('bold-vertical')`
4. GrapesJS canvas renders with bold-vertical placeholder HTML at 860px width
5. Canvas surround is `#1e293b` (dark slate). Template frame has white background with rounded corners and shadow.

### CSS Accumulation Fix (Success Criterion 2)

Not a visible interaction — internal idempotency fix in `injectHeadResources()`. The visual contract: after 5 reloads, `editor.getCss().length` does not grow. No new UI chrome needed to communicate this to the user.

Implementation approach: Claude's discretion (D-03). Suggested: check for `data-gjs-injected` attribute on style tags before re-injection, or clear existing injected styles before re-injecting.

### OneShot Removal (CLEAN-01)

Not a user-visible interaction. Post-removal, `packages/templates/src/index.ts` must not export any oneshot identifiers. Verification: `grep -r "oneshot" apps/web packages/` returns no results.
