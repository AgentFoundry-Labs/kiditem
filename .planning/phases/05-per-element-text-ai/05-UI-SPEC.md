## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (e.g. `p-0.5`, `gap-1`) |
| sm | 8px | Compact element spacing (e.g. `px-2`, `py-2`, `gap-2`) |
| md | 16px | Default element spacing (e.g. `px-3 py-2` areas) |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Panel width is fixed at 280px (matching `AIImageEditPanel` w-[280px]). Panel minimum touch target height for preset buttons: 32px (`py-2` + text height). Custom textarea: `rows={3}` (~48px tall).

Source: `AIImageEditPanel.tsx` — all spacing tokens verified directly from existing panel component.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 |
| Label / UI element | 12px (text-xs) | 700 (bold) | 1.4 |
| Display | not used in this phase | — | — |

All text in the `AITextEditPanel` uses `text-xs` (12px) with `font-bold`, matching the `AIImageEditPanel` panel header pattern. This applies to: the panel header title, preset button labels, loading banner text, error text, custom toggle label, custom submit button label. Body-level copy (14px `font-normal`) is not used inside this panel.

Exactly 2 weights: `font-normal` (400) for general body text outside the panel; `font-bold` (700) for all labeled UI elements inside the panel.

Source: `AIImageEditPanel.tsx` — `text-xs font-bold` class verified on panel header. Weight count reduced from 3 to 2 per spec constraint (removed `font-medium` / 500).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #ffffff | Panel background (`bg-white`), preset button resting state (`bg-gray-50`) |
| Secondary (30%) | #f8fafc (gray-50) / #f1f5f9 (gray-100) | Canvas surrounding the panel, hover states on neutral actions |
| Accent (10%) | #10b981 (emerald-500) | AI action submit button, loading banner background tint, focus ring on text inputs |
| Destructive | not applicable in Phase 5 | No destructive actions |

Accent reserved for: (1) the primary submit button for custom prompt ("AI 텍스트 적용"), (2) the loading state banner (`bg-emerald-50` + `text-emerald-700`), (3) focus ring on textarea and text input (`focus:ring-emerald-500`). Accent is NOT used for preset buttons in their resting state.

Preset button hover states use accent-family color to indicate active selection:
- 다시쓰기 / 번역 / 축약 buttons: `hover:bg-emerald-50 hover:text-emerald-700` (matching AIImageEditPanel)
- Panel border and dividers: `border-gray-200` / `border-gray-100`
- Error text: `text-red-500` (semantic red, not accent)

Source: `AIImageEditPanel.tsx` — emerald palette verified. `globals.css` — `--accent: #2563eb` (blue-600) is the global accent token, but the editor panels use emerald independently for AI actions. This phase continues the emerald convention established by `AIImageEditPanel`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "AI 텍스트 적용" |
| Panel header title | "텍스트 편집" |
| Preset button — rewrite | "다시쓰기" |
| Preset button — translate | "번역" |
| Preset button — shorten | "축약" |
| Loading banner | "AI 변환 중..." |
| Custom prompt toggle | "직접 입력" |
| Custom prompt placeholder | "원하는 변환 내용을 입력하세요..." |
| Custom prompt submit button | "AI 텍스트 적용" |
| Error state | "변환에 실패했습니다. 다시 시도해 주세요." (auto-dismisses after 4 seconds — no manual dismiss needed) |
| Empty custom prompt guard | input disabled state — no copy needed, button disabled |
| isBusy guard | no visible copy — preset buttons enter `disabled` + `opacity-50` state silently |

No empty state: the panel appears only when a text element is selected. If no text element is selected the panel is absent (no empty state to display).

No destructive actions in Phase 5. No confirmation dialog needed.

Source: Decisions D-01 through D-04 from `05-CONTEXT.md`. Korean copy convention follows `AIImageEditPanel.tsx` ("AI 편집 중...", "편집에 실패했습니다", "직접 입력"). Adapted to text context. Error copy updated to include solution hint per non-blocking FLAG-3.

### AITextEditPanel Layout Spec

Focal point: the three preset buttons (다시쓰기, 번역, 축약) are the primary affordance. They occupy the visual center of the panel and receive the most prominent placement. The custom prompt section is secondary, collapsed by default.

```
[Panel — 280px wide, bg-white, rounded-xl, shadow-lg, border border-gray-200]
├── Header row — px-3 py-2, border-b border-gray-100
│   ├── Left: <Wand2 size={14} /> + "텍스트 편집" (text-xs font-bold text-gray-700)
│   └── Right: X close button (p-0.5, text-gray-400 hover:text-gray-600,
│       aria-label="닫기")
│
├── [Conditional] Loading banner — px-3 py-2, bg-emerald-50, border-b border-emerald-100
│   └── <Loader2 size={14} animate-spin text-emerald-600 /> + "AI 변환 중..." (text-xs font-bold text-emerald-700)
│
├── Preset buttons section — px-3 pt-3 pb-1, space-y-1
│   ├── "다시쓰기" button — <Wand2 size={14} />, full-width, bg-gray-50 hover:bg-emerald-50 hover:text-emerald-700, text-xs font-bold, rounded-lg, py-2
│   ├── "번역" button     — <Languages size={14} />, same style
│   └── "축약" button     — <AlignLeft size={14} />, same style
│   All buttons: disabled state = opacity-50 cursor-not-allowed (during isBusy or loading)
│   Loading preset: show <Loader2 size={14} animate-spin /> replacing icon
│
├── Custom section — px-3 pb-3
│   ├── Toggle: "직접 입력" (text-xs text-gray-500 hover:text-gray-700) + ChevronDown/Up icon
│   └── [Expanded]
│       ├── textarea rows={3} — text-xs, border-gray-200, rounded-lg, resize-none,
│       │   focus:ring-1 focus:ring-emerald-500, placeholder "원하는 변환 내용을 입력하세요..."
│       └── Submit button — "AI 텍스트 적용", text-xs font-bold, text-white,
│           bg-emerald-500 hover:bg-emerald-600, rounded-lg, w-full py-2,
│           disabled when loading or empty prompt
│
└── [Conditional] Error row — px-3 pb-2
    └── "변환에 실패했습니다. 다시 시도해 주세요." (text-xs text-red-500 font-bold), auto-dismissed after 4s
```

Icons from lucide-react: `Wand2` (다시쓰기), `Languages` (번역), `AlignLeft` (축약), `Send` (custom submit alternative if Wand2 is used in header), `X` (close), `ChevronDown`/`ChevronUp` (custom toggle), `Loader2` (loading spinner).

Source: `AIImageEditPanel.tsx` — layout mirrored exactly. Icons chosen to semantically match Korean preset labels. D-05 from `05-CONTEXT.md` — panel appears below the selected element, Notion AI style.

## Interaction States

| State | Trigger | Visual |
|-------|---------|--------|
| Panel hidden | No text element selected / deselect | Portal unmounts, spot removed |
| Panel visible | Text element selected via component:selected | Portal renders at spot position below element |
| Panel repositions | Canvas scroll / zoom / resize | canvas:spot event fires, spotStyle updates, panel follows |
| Preset idle | Default | bg-gray-50 text-gray-700 |
| Preset hover | Mouse over | bg-emerald-50 text-emerald-700 (transition-colors) |
| Preset loading | AI request in-flight for this preset | Loader2 replaces icon, animate-spin |
| All buttons disabled | isBusy = true or loading = true | opacity-50 cursor-not-allowed |
| Loading banner | loading = true | Shown below header, bg-emerald-50 |
| Error | API call throws | Error row shown at bottom, auto-dismiss 4s |
| Custom collapsed | Default | Only toggle button visible |
| Custom expanded | Toggle clicked | Textarea + submit button revealed |
| Undo | Cmd+Z after AI apply | Single step reverts to original text (GrapesJS UndoManager) |
