# KidItem Docs

`docs/` is for durable project documentation only: current architecture,
testing, generated ERD/Graphify navigation, dev data guidance, and release
notes that explain shipped behavior.

Do not accumulate session plans, scratch specs, agent logs, or temporary
coordination notes here. Keep those in local `.omx/` / `.omc/` scratch space
and promote only enduring rules into the nearest scoped `AGENTS.md`.

Generated navigation docs:

- `docs/ERD.md` and `docs/erd/*.md` are generated by `npm run graphify:schema`.
- `docs/GRAPHIFY.md` documents the committed Graphify outputs under
  `graphify-out/`.
