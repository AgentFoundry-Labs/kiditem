# References

`docs/references/` contains durable external reference material: vendor PDFs,
spreadsheets, and source documents that support implementation decisions.

Rules:

- Do not put generated outputs or session scratch files here.
- Prefer linking official online docs from Markdown when the source is stable.
- Commit binary files only when the team needs the exact artifact offline.
- Do not store secrets, credentials, downloaded payload dumps, or private
  scraper bundle contents here.

Shared development data belongs in Google Drive `KidItem Dev Data`, not this
directory. See [DEV_DATA_BUNDLES.md](../DEV_DATA_BUNDLES.md).
